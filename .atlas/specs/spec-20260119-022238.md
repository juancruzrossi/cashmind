# Feature: Financial Health Assessment

## Overview
Financial assessment system that calculates a score based on 4 key metrics (savings rate, fixed expenses ratio, budget adherence, monthly trend) and presents it in a traffic light format (Red/Yellow/Green). Includes personalized advice generated by Gemini and a visual history of monthly evolution.

## Requirements

### Functional
- FR-1: Calculate a financial health score based on 4 weighted metrics
- FR-2: Show an individual traffic light for each metric plus a global traffic light
- FR-3: Generate personalized advice with Gemini for red/yellow metrics
- FR-4: Save monthly score snapshots for history
- FR-5: Show a score evolution chart over time
- FR-6: Dashboard widget with the global traffic light (visual only)
- FR-7: Dedicated section with full breakdown and advice
- FR-8: Guided onboarding when there is not enough data

### Non-Functional
- NFR-1: Score is calculated when opening the section (not in real time)
- NFR-2: Gemini advice is cached until the user clicks "regenerate"
- NFR-3: API must respond in <500ms for score calculation

## Technical Design

### Approach
- Backend: New `/api/health-score/` endpoint that calculates metrics and score
- Backend: New `HealthScoreSnapshot` model for monthly history
- Backend: `/api/health-score/advice/` endpoint for Gemini advice
- Frontend: New `/health` section with full detail
- Frontend: `HealthScoreWidget` widget for the dashboard

### Data Model

```python
class HealthScoreSnapshot(models.Model):
    user = models.ForeignKey(User, on_delete=models.CASCADE)
    month = models.DateField()  # First day of the evaluated month

    # Individual scores (0-100)
    savings_rate_score = models.IntegerField()
    fixed_expenses_score = models.IntegerField()
    budget_adherence_score = models.IntegerField()
    trend_score = models.IntegerField()

    # Overall score (0-100)
    overall_score = models.IntegerField()

    # Traffic light status: 'red', 'yellow', 'green'
    overall_status = models.CharField(max_length=10)

    # Cached Gemini advice
    cached_advice = models.TextField(null=True, blank=True)
    advice_generated_at = models.DateTimeField(null=True)

    created_at = models.DateTimeField(auto_now_add=True)

    class Meta:
        unique_together = ['user', 'month']
```

### Metric Calculation

**1. Savings Rate (savings_rate)**
```
savings_rate = (income - expenses) / income * 100
Green: >= 20%
Yellow: 10-19%
Red: < 10% or no income
```

**2. Fixed Expenses Ratio (fixed_expenses_ratio)**
```
Fixed categories: housing, services, transportation, insurance
fixed_ratio = fixed_expenses / income * 100
Green: <= 40%
Yellow: 41-55%
Red: > 55%
```

**3. Budget Adherence (budget_adherence)**
```
budgeted_categories = categories with a defined Budget
compliant_categories = categories where expense <= limit
adherence = compliant_categories / budgeted_categories * 100
Green: >= 80%
Yellow: 50-79%
Red: < 50% or no budgets defined
```

**4. Monthly Trend (trend)**
```
Compare current month expenses vs previous month
improvement = (previous_expenses - current_expenses) / previous_expenses * 100
Green: improvement >= 5% or stable expenses
Yellow: worsens 0-10%
Red: worsens > 10%
```

**Overall Score**
```
overall = (savings*30 + fixed*25 + budget*25 + trend*20) / 100
Green: >= 70
Yellow: 40-69
Red: < 40
```

## UX/UI

### Main User Flow
1. User navigates to Dashboard
2. Sees widget with the global financial health traffic light
3. Clicks "View details" (link, not the widget)
4. Sees section with 4 metric cards (each with a traffic light)
5. Sees monthly evolution chart
6. Sees advice panel with "Regenerate advice" button

### Onboarding (not enough data)
1. User opens the financial health section
2. System detects < 1 month of transactions
3. Shows wizard: "To assess your financial health we need:"
   - At least 1 registered income
   - At least 5 expense transactions
   - Budgets defined for at least 3 categories
4. Buttons to navigate and register each missing item

### Error States
- No income in the month: traffic light automatically RED with explanatory message
- Gemini error: show generic predefined advice + retry option
- No history: show only the current assessment without chart

### UI Components

**Dashboard Widget (HealthScoreWidget)**
- Circle with traffic light (green/yellow/red)
- Text: "Financial Health: [Status]"
- No interaction (separate "View details" link)

**Health Section (/health)**
- Header with large global traffic light + numeric score
- 2x2 grid of MetricCards:
  - Savings Rate (piggy bank icon)
  - Fixed Expenses (home icon)
  - Budget (target icon)
  - Trend (trending icon)
- Evolution chart (line chart, last 6-12 months)
- Advice panel with regenerate button

**MetricCard**
- Small traffic light
- Metric name
- Current value (e.g. "23%")
- Visual progress bar

## Interview Summary

**Key decisions:**
- 3-level traffic light format (Red/Yellow/Green) for simplicity
- All 4 metrics included: savings, fixed expenses, budget, trend
- Optimal savings rate: 20% (50/30/20 rule)
- Maximum healthy fixed expenses: 40% (includes housing, services, transportation, insurance)
- AI-generated advice (Gemini) with cache and a manual regenerate button
- Backend calculation for consistency and easier historical tracking
- History with monthly evolution chart
- Visual-only widget (no click), separate link for details
- No income = automatically red (no workarounds)
- Guided onboarding for new users without enough data
